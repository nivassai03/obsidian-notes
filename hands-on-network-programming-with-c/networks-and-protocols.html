Networks and Protocols

TCP/IP layer model

The TCP/IP protocol suite is the most common network communication model in use today.

Network Access Layer

On this layer physical connections and data framing happens. Sending an ethernet or wifi packet are examples of this layer 1 concerns.

Internet Layer

This layer deals with concerns of addressing packets and routing them over multiple interconnection networks. Its at this layer IP address is defined.

Host-to-Host Layer

It provided two protocols TCP and UDP. These protocols address concerns such as data order, data segmentation, network congestion and error correction.

Process / Application Layer

The process/application layer is where the protocols such as HTTPS,SMTP and FTP are implemented.

Data Encapsulation

  • Text is encoded in the html structure.
  • Html is transmitted as part Http response.
  • Http is transmitted as a part of Tcp session.
  • Tcp packet is routed by an IP packet.
  • IP packet is transmitted over the wire in an Ethernet packet (or another protocol.)

Internet Protocol

It comes in two version IPv4 & IPv6.
IPv4 uses 32bit addressing, which limits its addressing to 2^32 (4,294,967,296) systems.
IPv6 uses 128bit addressing, which allows its addressing upto 2^128 (upto 3.4 x 1038) systems.

Address

All internet protocol traffic routes to an address.
IPv4 are 32 bit. They are commonly divided into four 8-bit sections. Ex.

  • 0.0.0.0

  • 127.0.0.1

  • 10.0.0.0

  • 172.16.0.5

  • 192.168.0.1

  • 192.168.50.1

  • 255.255.255.255

    127.0.0.1 is a loopback address. This essentially mean establish a connection to myself.
    IPv4 reserves some address rangs for private use.

  • 10.0.0.0 to 10.255.255.255

  • 172.16.0.0 to 172.31.255.255

  • 192.168.0.0 to 192.168.255.255

Using classless Inter-Domain Routing (CIDR) notation, we can write the three previous address range as follows. CIDR notation works by specifying the number of bits that are fixed.

  • 10.0.0.0/8 (specifies first 8 bits are fixed.)
  • 176.16.0.0/12
  • 192.168.0.0/16

IPv6 address are 128 bit long. They are written as eight groups of four hexadecimal characters delineated by columns. Standard is to use lower case letters in IPv6 address.

  • 0000:0000:0000:0000:0000:0000:0000:0001
  • 2001:0db8:0000:0000:0000:ff00:0042:8329
  • fe80:0000:0000:0000:75f4:ac69:5fa7:67f9
  • ffff:ffff:ffff:ffff:ffff:ffff:ffff:ffff

::1 is the loopback address for IPv6.

Rules for shortening IPv6 address.

  • Allows for leading zeros to be omitted.
  • Allows for consecutive sections of zeros to be replaced with (::). It can only be used once.
    • ::1
    • 2001:db8::ff00:42:8329
    • fe80::75f4:ac69:5fa7:67f9
    • ffff:ffff:ffff:ffff:ffff:ffff:ffff:ffff

Dual stack implementations also recognise a special class of IPv6 that map directly to the IPv4. These starts with 80 zero bits and then by 16 one bits followed by 32 bit IPv4 address.

IPv6 Address Mapped IPv4 Address
::ffff:10.0.0.0 10.0.0.0
::ffff:172.16.0.5 172.16.0.5
::ffff:192.168.0.1 192.168.0.1
::ffff:192.168.50.1 192.168.50.1

Link Local Address

Are only usable on the local link. Routers never forward packets from these address.
They are useful for a system to access auto configuration function before having an assigned IP address.

  • IPv4 (169.254.0.0/16) and IPv6 (fe80::/10)

Domain Names

To resolve a hostname your computer sends a UDP message to your DNS server and ask it for an AAAA-type record for the domain you are trying to resolve. If this record exists, an IPv6 address is returned. If no AAAA record exists, then your computer queries the server again, but asks for an A record. If this record exists, you will receive an IPv4 address for the server

Internet Routing

When you make a connection over the internet, your data first transmits to your local router. From there it is transmitted to another router, which is connected to another router, and so on.

tracert window utility list routers between your system and destination.
traceroute for unix systems.
When IP packets are transferred between networks their address must be translated.

Local Networks and Address Translation

Local Area Network (LANs) are IPv4 address range served for use in these small local networks. The private ranges are as follows.

  • 10.0.0.0 to 10.255.255.255
  • 172.16.0.0 to 172.31.255.255
  • 192.168.0.0 to 192.168.255.255

When a packet originates from a device on IPv4 local network, it must undergo Network Address Translation before being routed to the internet.

A router that implements NAT remembers which local address a connection is established from. Devices on same LAN could directly address one another by their local address.

Any traffic to the internet must undergo NAT. Router does this by modifying source IP address from the original private LAN IP to its public Internet IP address. Likewise, when the router receives the return communication, it must modify the destination address from its public IP to the private IP of the original sender

Subnetting and CIDR

IP address can be split into two parts. The most significant bits are used to identify the network or subnetwork, and the least significant are used to identify the specific device on the internet.

IPv4 traditionally uses a mask notation to identify the IP address parts. Router performs bitwise AND with subnet mask.
