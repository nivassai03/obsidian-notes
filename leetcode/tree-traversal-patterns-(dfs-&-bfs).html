Tree Traversal Patterns (DFS & BFS)

Recursive Preorder

Binary Tree Paths

Intitution:
We need to return all paths from root to leaf as strings. A path is valid only if it ends at a leaf node (node with no children). Since a tree branches, we must explore all possible paths → this naturally suggests DFS.

Key observation:

  • Each path is built incrementally while traversing
  • A path is complete only when we reach a leaf
  • At each node, we branch into left and right → explore both

Approach:

  • Use DFS (recursive)
  • Maintain a string path that represents current path
  • At each node:
    • Append current node value to path
  • If it is a leaf:
    • Push path into result
  • Otherwise:
    • Add "->"
    • Recurse left and right

Invariant:

  • path always contains the path from root → current node
  • When reaching a leaf, path is a valid complete answer

Edge cases (if any):

  • Empty tree → return empty vector
  • Single node → return [node value as string]

Time & Space complexity:

  • Time: O(N * H)
    • N = number of nodes
    • H = height of tree (string building cost)
  • Space: O(H)
    • recursion stack + path string

Code

class Solution {
    void dfs(TreeNode* root, string path, vector<string>& result)
    {
        if(root == nullptr) return;
        path += to_string(root->val);
        if(root->left == nullptr && root->right == nullptr)
        {
            result.push_back(path);
            return;
        }
        path += "->";
        dfs(root->left, path, result);
        dfs(root->right, path, result);
    }

public:
    vector<string> binaryTreePaths(TreeNode* root) {
        vector<string> result;
        dfs(root, "", result);
        return result;
    }
};

Symmetric Tree

Intitution:
We need to check if a binary tree is symmetric (mirror of itself). Instead of checking symmetry within one subtree, we compare two subtrees (left and right) simultaneously in a mirrored way.

Key observation:

  • Symmetry means left subtree is a mirror of right subtree
  • Mirror condition:
    • Values must be equal
    • Left child of one must match right child of the other
    • Right child of one must match left child of the other

Approach:

  • Define a recursive function isMirror(a, b)
  • Base cases:
    • If both are null → true
    • If one is null → false
  • Check:
    • Values equal
    • a->left with b->right
    • a->right with b->left
  • Call this function with (root->left, root->right)

Invariant:

  • At every step, we are comparing nodes that should be mirror positions
  • If all such pairs satisfy mirror condition → tree is symmetric

Edge cases (if any):

  • Empty tree → symmetric
  • Single node → symmetric
  • Skewed tree → not symmetric

Time & Space complexity:

  • Time: O(N)
    • Each node is visited once
  • Space: O(H)
    • Recursion stack (H = height of tree)
class Solution {
    bool isMirror(TreeNode* left, TreeNode* right)
    {
        if(left == nullptr && right == nullptr) return true;
        if(left == nullptr || right == nullptr) return false;

        return (left->val == right->val) &&
               isMirror(left->left, right->right) &&
               isMirror(left->right, right->left);
    }

public:
    bool isSymmetric(TreeNode* root) {
        if(root == nullptr) return true;
        return isMirror(root->left, root->right);
    }
};

Flatten Binary Tree to Linked List

Intitution:
We need to convert a binary tree into a linked list in-place following preorder traversal (root → left → right).
Final structure should use only right pointers (like a singly linked list).

Key observation:

  • Preorder means:
    current → left subtree → right subtree
  • If we move left subtree to right, we must not lose original right subtree
  • So we attach original right subtree to the rightmost node of left subtree

Approach:

  • Use a pointer cur to traverse the tree
  • For each node:
    • If left exists:
      • Find rightmost node of left subtree (prev)
      • Connect prev->right = cur->right
      • Move left subtree to right → cur->right = cur->left
      • Set cur->left = nullptr
    • Move to next node → cur = cur->right

Invariant:

  • At any point:
    • All nodes left of cur are already flattened
    • Structure from root to cur follows preorder
    • No node processed has a non-null left pointer

Edge cases (if any):

  • Empty tree → return
  • Single node → already flattened
  • Only left subtree → becomes right chain
  • Only right subtree → unchanged

Time & Space complexity:

  • Time: O(n) (each node visited once)
  • Space: O(1) (in-place, no recursion or stack)
class Solution {
public:
    void flatten(TreeNode* root) {
        TreeNode* cur = root;
        while (cur != nullptr) {
            if (cur->left != nullptr) {
                TreeNode* prev = cur->left;
                while (prev->right != nullptr) {
                    prev = prev->right;
                }
                prev->right = cur->right;
                cur->right = cur->left;
                cur->left = nullptr;
            }
            cur = cur->right;
        }
    }
};

Smallest String Starting From Leaf

Intitution:
We need to find the lexicographically smallest string formed from a leaf to the root.
Each node has a value [0–25] mapped to characters ['a'–'z'].
So every root-to-leaf path forms a string, but we must read it in reverse (leaf → root).

Key observation:

  • Paths are naturally formed root → leaf using DFS
  • But required string is leaf → root
  • Instead of reversing at leaf, build string by adding current character at the front
  • At every leaf, compare the formed string with current best

Approach:

  • Use DFS traversal
  • Maintain a string path
  • At each node:
    • Convert value to char → ch = node->val + 'a'
    • Prepend → path = ch + path
  • If leaf:
    • Compare with result and update if smaller
  • Recurse left and right
  • Use pass-by-value for path to avoid backtracking issues

Invariant:

  • At any node, path represents the string from current node → root
  • At leaf, path is a valid candidate answer (leaf → root)

Edge cases (if any):

  • Empty tree → return ""
  • Single node → return its character
  • Skewed tree → only one path exists

Time & Space complexity:

  • Time: O(n * h) (string operations up to height at each node)
  • Space: O(h) recursion stack

Code

class Solution {
    string result;
    void dfs(TreeNode* root, string path) {
        if (root == nullptr) return;
        char ch = root->val + 'a';
        path = ch + path;
        if (root->left == nullptr && root->right == nullptr) {
            if (result.empty() || path < result) {
                result = path;
            }
            return;
        }
        dfs(root->left, path);
        dfs(root->right, path);
    }

public:
    string smallestFromLeaf(TreeNode* root) {
        dfs(root, "");
        return result;
    }
};

Find Mode in Binary Search Tree

Intitution:

  • We need to find the most frequent value(s) in a BST
  • BST inorder traversal gives values in sorted order
  • In a sorted sequence, duplicates appear consecutively
  • So problem reduces to: count frequencies in sorted traversal and track max

Key observation:

  • Inorder traversal of BST → sorted order
  • Same values will be adjacent → easy frequency counting
  • We don’t need extra hashmap → can do in O(1) space (excluding recursion)

Approach:

  • Perform inorder traversal
  • Maintain:
    • prev → previous node
    • currCount → count of current value
    • maxCount → highest frequency seen so far
    • result → stores modes
  • For each node:
    • If prev exists and same value → currCount++
    • Else → currCount = 1
    • If currCount > maxCount:
      • clear result
      • add current value
      • update maxCount
    • If currCount == maxCount:
      • add current value
    • Update prev

Invariant:

  • At any node:
    • currCount = frequency of current value till now
    • maxCount = max frequency seen so far
    • result contains all values with frequency = maxCount

Edge cases (if any):

  • Empty tree → return empty list
  • All values same → single mode
  • All values unique → all are modes
  • Skewed tree → recursion depth increases

Time & Space complexity:

  • Time: O(n) (visit each node once)
  • Space: O(h) recursion stack
    • Balanced tree → O(log n)
    • Skewed tree → O(n)

Code

class Solution {
    TreeNode* prev = nullptr;
    int currCount = 0;
    int maxCount = 0;
    vector<int> result;

    void inorder(TreeNode* root) {
        if (root == nullptr) return;

        inorder(root->left);

        if (prev != nullptr && prev->val == root->val) {
            currCount++;
        } else {
            currCount = 1;
        }

        if (currCount > maxCount) {
            result.clear();
            result.push_back(root->val);
            maxCount = currCount;
        } else if (currCount == maxCount) {
            result.push_back(root->val);
        }
        prev = root;
        inorder(root->right);
    }

public:
    vector<int> findMode(TreeNode* root) {
        inorder(root);
        return result;
    }
};

Kth Smallest Element in a BST

Intitution:

  • We need to find the kth smallest element in a BST
  • BST property: inorder traversal gives sorted order
  • So kth smallest = kth element in inorder traversal

Key observation:

  • Inorder traversal (left → node → right) gives elements in increasing order
  • We don’t need full array, just count while traversing
  • Stop traversal once we reach kth element

Approach:

  • Perform inorder traversal
  • Maintain a counter cnt
  • For each node:
    • Traverse left
    • Increment cnt
    • If cnt == k → this node is answer
    • Traverse right (only if needed)
  • Use early stopping once answer is found

Invariant:

  • At any point:
    • cnt = number of nodes processed in inorder so far
    • When cnt == k → current node is kth smallest

Edge cases (if any):

  • k = 1 → smallest element (leftmost node)
  • k = n → largest element (rightmost node)
  • Skewed tree → behaves like sorted array
  • Tree with duplicates → counts duplicates as separate elements

Time & Space complexity:

  • Time: O(h + k)
    • In worst case: O(n)
  • Space: O(h) recursion stack
    • Balanced → O(log n)
    • Skewed → O(n)

Code

class Solution {
    bool found = false;

    void inorder(TreeNode* root,int& cnt,int& ans,int k)
    {
        if(root == nullptr || found) return;
        inorder(root->left,cnt,ans,k);
        if(found) return;
        cnt++;
        if(cnt == k)
        {
            ans = root->val;
            found = true;
            return;
        }
        inorder(root->right,cnt,ans,k);
    }

public:
    int kthSmallest(TreeNode* root, int k) {
        int cnt = 0;
        int ans = -1;
        inorder(root,cnt,ans,k);
        return ans;
    }
};

Validate Binary Search Tree

Intitution:

  • We need to check if a binary tree satisfies BST rules
  • Local comparison (parent vs immediate child) is NOT enough
  • Each node must satisfy constraints coming from all its ancestors
  • So every node must lie within a valid range (min, max)

Key observation:

  • In BST:
    • Left subtree → all values < root
    • Right subtree → all values > root
  • This rule applies recursively for entire subtree, not just immediate children
  • So each node has a valid range:
    • Left child range: (low, root->val)
    • Right child range: (root->val, high)

Approach:

  • Use recursion with range limits
  • Start with range (-inf, +inf)
  • For each node:
    • If node->val <= low OR node->val >= high → invalid
    • Recurse left with (low, node->val)
    • Recurse right with (node->val, high)
  • If all nodes satisfy range → valid BST

Invariant:

  • At any node:
    • low < node->val < high
    • All nodes in subtree must obey inherited constraints

Edge cases (if any):

  • Empty tree → valid BST
  • Single node → valid BST
  • Duplicate values → invalid (strict inequality required)
  • INT_MIN / INT_MAX → use long long to avoid overflow issues
  • Skewed tree → deep recursion

Time & Space complexity:

  • Time: O(n)
  • Space: O(h)
    • Balanced → O(log n)
    • Skewed → O(n)

Code

class Solution {
    bool validate(TreeNode* root, long long low, long long high) {
        if (root == nullptr) return true;

        if (root->val <= low || root->val >= high) {
            return false;
        }

        return validate(root->left, low, root->val) &&
               validate(root->right, root->val, high);
    }

public:
    bool isValidBST(TreeNode* root) {
        return validate(root, LLONG_MIN, LLONG_MAX);
    }
};

Recursive Inorder

Find Mode in Binary Search Tree

Intitution:

  • We need to find the value(s) that appear most frequently in a BST
  • BST inorder traversal gives values in sorted order
  • In sorted order, duplicates appear consecutively
  • So we can count frequencies while traversing inorder

Key observation:

  • Inorder traversal → sorted sequence
  • Same values appear adjacent
  • We only need:
    • previous node
    • current frequency
    • maximum frequency
  • No hashmap required

Approach:

  • Perform inorder traversal
  • Maintain:
    • prev → previous visited node
    • cnt → current value frequency
    • max_count → highest frequency seen
    • result → stores modes
  • For each node:
    • If same as prev:
      • cnt++
    • Else:
      • cnt = 1
    • Compare cnt with max_count:
      • cnt > max_count:
        • clear result
        • add current value
        • update max_count
      • cnt == max_count:
        • add current value
  • Update prev after processing node

Invariant:

  • During inorder traversal:
    • cnt = frequency of current value block
    • max_count = highest frequency encountered so far
    • result contains all values with frequency = max_count

Common mistake:

  • Updating frequency only when duplicate occurs
  • Every value must be processed at least once
  • When current value changes:
    • reset cnt = 1
  • Otherwise unique values never get counted

Wrong logic:

  • Only handling:
    • prev != nullptr && prev->val == root->val
  • This ignores first occurrence of values

Correct logic:

  • Same value:
    • cnt++
  • New value:
    • cnt = 1

Edge cases (if any):

  • Empty tree → return empty vector
  • Single node → node itself is mode
  • All values unique → all are modes
  • All values same → one mode
  • Multiple modes possible

Time & Space complexity:

  • Time: O(n)
  • Space: O(h)
    • Balanced tree → O(log n)
    • Skewed tree → O(n)

Code

class Solution {
    TreeNode* prev = nullptr;
    int cnt = 0;
    int max_count = 0;

    void dfs(TreeNode* root, vector<int>& result) {
        if (root == nullptr) return;

        dfs(root->left, result);

        if (prev != nullptr && prev->val == root->val) {
            cnt++;
        } else {
            cnt = 1;
        }

        if (cnt > max_count) {
            result.clear();
            result.push_back(root->val);
            max_count = cnt;
        } else if (cnt == max_count) {
            result.push_back(root->val);
        }

        prev = root;

        dfs(root->right, result);
    }

public:
    vector<int> findMode(TreeNode* root) {
        vector<int> result;
        dfs(root, result);
        return result;
    }
};

Kth Smallest Element in a BST

Intitution:

  • BST inorder traversal gives elements in sorted order
  • kth visited node during inorder traversal = kth smallest element
  • We only need a counter while traversing

Key observation:

  • Inorder traversal order:
    • left → node → right
  • Nodes are visited in increasing order
  • Count nodes during traversal
  • When count becomes k → current node is answer

Approach:

  • Perform inorder traversal
  • Maintain:
    • cnt → number of visited nodes
    • result → kth smallest value
  • For every visited node:
    • cnt++
    • if cnt == k:
      • store answer
      • stop traversal early

Invariant:

  • During traversal:
    • cnt = number of nodes processed in sorted order
    • When cnt == k:
      • current node is kth smallest

Correct flow:

  • traverse left
  • cnt++
  • if cnt == k:
    • answer found
  • traverse right

Early stopping optimization:

  • Once kth element found:
  • stop further recursion
  • Avoid unnecessary traversal

Edge cases (if any):

  • k = 1 → smallest element
  • k = n → largest element
  • Single node tree
  • Skewed BST behaves like sorted linked list
  • Duplicates count as separate elements

Time & Space complexity:

  • Time:
    • Average: O(h + k)
    • Worst: O(n)
  • Space: O(h)
    • Balanced tree → O(log n)
    • Skewed tree → O(n)

Code

class Solution {
    int cnt = 0;

    void inorder(TreeNode* root, int k, int& result)
    {
        if (root == nullptr || cnt >= k) return;
        inorder(root->left, k, result);
        cnt++;
        if (cnt == k) {
            result = root->val;
            return;
        }

        inorder(root->right, k, result);
    }

public:
    int kthSmallest(TreeNode* root, int k) {
        int result = -1;
        inorder(root, k, result);
        return result;
    }
};

Binary Search Tree Iterator

Intitution:

  • BST inorder traversal gives elements in sorted order
  • BST Iterator should return elements one-by-one in sorted order
  • Instead of storing entire inorder traversal beforehand, we can simulate inorder traversal lazily using a stack

Key observation:

  • In recursive inorder traversal:
    • recursion stack stores path to current node
  • We can manually simulate this recursion stack using explicit stack
  • The next smallest element is always the top of the stack

Approach:

  • Maintain a stack
  • Initially:
    • push all left nodes from root
  • Why?
    • Leftmost node is smallest element
  • For next():
    • pop top node
    • this is next smallest
    • if popped node has right child:
      • push all left nodes of right subtree
  • For hasNext():
    • return whether stack is empty or not

Invariant:

  • Stack always stores path to next inorder node
  • Top of stack = next smallest unvisited element

Why pushing left chain works:

  • Inorder traversal:
    • left → node → right
  • So before visiting any node:
    • we must completely process left subtree
  • Therefore:
    • always move as left as possible first

Why this is optimal:

  • We do traversal only when needed
  • Called lazy inorder traversal
  • Avoids storing entire tree traversal

Comparison with brute force:

  • Brute force:

    • store entire inorder traversal in vector
    • space: O(n)

  • Optimal:

    • store only root-to-leaf path
    • space: O(h)

Complexities:

  • Constructor:

    • O(h)

  • next():

    • Average/amortized: O(1)
    • Worst case: O(h)

  • hasNext():

    • O(1)

  • Space:

    • O(h)
    • h = tree height

Edge cases:

  • Empty tree
  • Single node tree
  • Completely skewed tree
  • Right-heavy tree
  • Left-heavy tree

Code

class BSTIterator {
    stack<TreeNode*> st;

    void pushLeft(TreeNode* root) {
        while (root != nullptr) {
            st.push(root);
            root = root->left;
        }
    }

public:
    BSTIterator(TreeNode* root) {
        pushLeft(root);
    }

    int next() {
        TreeNode* node = st.top();
        st.pop();

        if (node->right != nullptr) {
            pushLeft(node->right);
        }

        return node->val;
    }

    bool hasNext() {
        return !st.empty();
    }
};
