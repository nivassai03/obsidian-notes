        ListNode* ptr1 = head;

Remove Nth Node From The End Of List

Intituion
The key intuition behind this problem is to avoid counting from the end directly and instead maintain a fixed distance between two pointers so that their relative position reveals the answer in one pass. We start by moving a fast pointer n steps ahead of a slow pointer, which creates a gap of n nodes between them. Then we move both pointers forward one step at a time while preserving this gap; this is the crucial invariant. By the time the fast pointer reaches the last node of the list, the slow pointer will be exactly one node before the node we need to delete, because the gap has been maintained throughout the traversal. This works because the fast pointer effectively “measures” how far we are from the end, and the slow pointer lags behind by n nodes, aligning it perfectly with the target’s predecessor. We stop when fast->next == nullptr (i.e., fast is at the last node) so that slow is positioned correctly without overshooting, and then we can safely remove slow->next. Using a dummy node before the head simplifies edge cases like deleting the first node, ensuring the same logic works uniformly for all cases.

Code

class Solution {
public:
    ListNode* removeNthFromEnd(ListNode* head, int n) {
        ListNode dummy(0,head);
        ListNode* sp = &dummy;
        ListNode* fp = &dummy;
        for(int i=0;i<n;i++)
        {
            fp = fp->next;
        }
        while(fp->next != nullptr)
        {
            sp = sp->next;
            fp = fp->next;
        }

        ListNode* del = sp->next;
        sp->next = del->next;
        delete del;
        return dummy.next;
    }
};

Delete the Middle Node of a Linked List

The key idea behind this problem is to identify the middle of a singly linked list without explicitly computing its length, and then remove that node efficiently. To achieve this in one pass, we use the two-pointer technique, where pointers move at different speeds to simulate positional relationships within the list.

We introduce a Slow Pointer that moves one step at a time and a Fast Pointer that moves two steps at a time. Because the fast pointer advances twice as quickly, it traverses the entire list while the slow pointer only covers half the distance. As a result, when the fast pointer reaches the end of the list, the slow pointer naturally lands at the middle node.

Invariant: At any point during traversal, the fast pointer has covered approximately twice the distance of the slow pointer. This ensures that when fast reaches the end (null), slow is positioned at index ⌊n/2⌋.

However, there is an important constraint in singly linked lists:
to delete a node, we must have access to its previous node, since deletion requires updating prev->next.

This introduces a challenge — the slow pointer directly points to the middle node, but we don’t have access to its predecessor.

To solve this, we maintain an additional pointer, prev, which always trails one step behind the slow pointer. As the traversal progresses:

  • slow moves toward the middle
  • prev tracks the node just before slow

By the time the fast pointer reaches the end:

  • slow points to the middle node
  • prev points to the node before the middle
	// Using Prev
    ListNode* deleteMiddle(ListNode* head) {
        if (!head || head->next == nullptr)
            return nullptr;
        ListNode* sp = head;
        ListNode* fp = head;
        ListNode* prev = nullptr;

        while (fp != nullptr && fp->next != nullptr) {
            prev = sp;
            sp = sp->next;
            fp = fp->next->next;
        }

        prev->next = sp->next;
        delete sp;
        return head;
    }
    // Using Dummy
    ListNode* deleteMiddle(ListNode* head) {

        if (head == nullptr || head->next == nullptr)
            return nullptr;
        ListNode dummy(0, head);
        ListNode* sp = &dummy;
        ListNode* fp = head;
        while (fp != nullptr && fp->next != nullptr) {
            sp = sp->next;
            fp = fp->next->next;
        }
        ListNode* del = sp->next;
        sp->next = del->next;
        delete del;

        return dummy.next;
    }

String Comparison with Special Characters

Backspace String Compare

Intitution

The fundamental challenge in this problem is that the # character represents a backspace operation, which removes the most recently added valid character. This behavior is inherently Last-In-First-Out (LIFO), which naturally maps to the concept of a stack.

In the stack approach, we simulate typing the string character by character. For every normal character, we push it onto the stack, representing that it is part of the current valid string. When we encounter a #, we pop from the stack, which removes the most recent character, mimicking the backspace operation. By the end of the traversal, the stack contains the final processed version of the string. We repeat this process for both input strings and then compare the resulting stacks (or constructed strings). This approach is straightforward and closely mirrors the actual behavior of typing with backspaces, but it requires O(n) extra space.

In the optimized approach, we treat the string as a writable buffer and maintain a pointer k that represents the “top of the simulated stack.” As we traverse the string with another pointer i, each normal character is written at position k, and k is incremented, effectively simulating a push operation. When a # is encountered, we simply decrement k (if possible), which simulates a pop operation by discarding the last valid character. At any point, the substring s[0...k-1] represents the processed string after applying all backspaces up to index i. This eliminates the need for an external stack while preserving the same logic. After processing both strings in this way, we compare their valid prefixes to determine if they are equal.

Code

    // Stack based
    bool backspaceCompare(string s, string t) {
        std::stack<char> s1;
        std::stack<char> s2;
        for (char c : s) {
            if (c != '#')
                s1.push(c);
            else {
                if (!s1.empty())
                    s1.pop();
            }
        }
        for (char c : t) {
            if (c != '#')
                s2.push(c);
            else {
                if (!s2.empty())
                    s2.pop();
            }
        }
        if (s1.size() != s2.size())
            return false;
        int n = s1.size();
        for (auto i = 0; i < n; ++i) {
            if (s1.top() != s2.top())
                return false;
            s1.pop();
            s2.pop();
        }
        return true;
    }
    // Optimized approach with stack concept
    bool backspaceCompare(string s, string t) {
        int k = 0, p = 0;
        for (int i = 0; i < s.size(); i++) {
            if (s[i] == '#') {
                k--;
                k = max(0, k);
            } else {
                s[k] = s[i];
                ++k;
            }
        }
        for (int i = 0; i < t.size(); i++) {
            if (t[i] == '#') {
                p--;
                p = max(0, p);
            } else {
                t[p] = t[i];
                ++p;
            }
        }
        if (k != p)
            return false;
        for (int i = 0; i < k; ++i) {
            if (s[i] != t[i])
                return false;
        }
        return true;

Expanding From Center

Palindromic Substrings

Intuition

The fundamental observation in this problem is that a palindrome is symmetric around its center. Instead of checking all substrings, we can leverage this symmetry by expanding outward from a center. Every palindrome must originate from a valid center, and expanding from that center generates all possible palindromes efficiently.

In this approach, we treat each position (and each gap between positions) as a potential center. From each center, we expand outward as long as the characters on both sides match. Each successful expansion represents a valid palindrome.

There are two types of centers:

  • A single character center (for odd-length palindromes)
  • A gap between characters (for even-length palindromes)

By iterating over all possible centers and expanding, we ensure that every palindrome is counted exactly once.

During expansion, the substring defined by the current left and right pointers is always a palindrome.

This holds because:

  • We only expand when the characters at both ends match
  • The inner substring was already confirmed to be a palindrome in the previous step

Code

class Solution {
public:
    int expandAndCount(const string& s, int left, int right) {
        int n = s.size();
        int cnt{0};
        while (left >= 0 && right < n && s[left] == s[right]) {
            ++cnt;
            --left;
            ++right;
        }
        return cnt;
    }
    int countSubstrings(string s) {
        int n = s.length();
        int result{0};
        for (int i = 0; i < n; ++i) {
            result += expandAndCount(s,i,i);
            result += expandAndCount(s,i,i+1);
        }
        return result;
    }
}
